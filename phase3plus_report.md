## 3. Extended Thinking Implementation

Both RSA and Stress Tester use the two-step pattern:

| Agent | Step A (Thinking) | Step B (Extraction) |
|-------|-------------------|---------------------|
| RSA | `thinking.budget_tokens=10000`, max_tokens=16000 | tool_choice=rsa_solution, max_tokens=4096 |
| Stress Tester | `thinking.budget_tokens=12000`, max_tokens=20000 | tool_choice=stress_test_result, max_tokens=4096 |

**Pattern:** Step A uses extended thinking (no tools), model reasons through the problem. Step B passes Step A output to a structured extraction call with tool_choice. This is necessary because `tool_choice` and `thinking` cannot be combined in a single Anthropic API call.

## 4. Deviations from SOW Spec

## 5. Guardrail Compiler Pattern Coverage

| Pattern Type | Patterns | Classification |
|-------------|----------|----------------|
| Obligations | `must`, `shall`, `is required to`, `obliged to`, `has a duty to` | hard_constraint (BINDING) |
| Prohibitions | `must not`, `shall not`, `is prohibited from`, `may not`, `is not permitted to` | hard_constraint (BINDING) |
| Thresholds | Numeric + regulatory units (%, days, Tier N Capital, etc.) | Extracted separately |
| Exceptions | `except`, `unless`, `provided that`, `subject to`, `notwithstanding` | Extracted separately |
| **Guidance** | `should`, `is expected to`, `is encouraged to`, `best practice` | soft_constraint |
| **Permissive** | `may`, `is permitted to`, `at its discretion` | Extracted separately |
| **Conditional** | `if/where/when...must/shall` | hard_constraint (BINDING) |
